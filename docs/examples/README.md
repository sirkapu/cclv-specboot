# Examples

Reference examples of project-specific customizations of the cclv-specboot kit. Each shows what one team's adaptation looks like — useful as a starting point or a comparison when you're customizing for your own project.

## Index

| Example | What it demonstrates |
|---------|---------------------|
| [`aura-style-bootstrap.md`](./aura-style-bootstrap.md) | AURA-flavored paste-prompt with credits system, desktop-first, light Portal/Translate safety baked in. Shows the §19 "Project-Specific Patterns" section in action. |

## How to use these

- **Skim before customizing your own.** Examples make the abstract patterns concrete.
- **Copy + adapt.** Take what fits your project; drop what doesn't.
- **Don't paste an example directly into a new project.** The canonical paste-prompt is `BOOTSTRAP-PROMPT.md` at the repo root, which leaves project-specific decisions as TBDs. Examples are illustrative.

## Contributing examples

If you've customized cclv-specboot for your project and want to share the result, see `CONTRIBUTING.md` at the repo root. Examples should be project-anonymized where possible (no real secrets, no real customer data).
